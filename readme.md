# Запуск

Перед запуском приложения необходимо задать в файле `appsetting.json` в секции `ConnectionStrings:TasksDatabase` строку подключения, соответствующую вашему серверу PostgreSQL.

Для получения логов требуется установить сервер логгирования SEQ и указать его адрес в секции `Serilog:Seq` в этом же файле. При отсутствии SEQ приложение будет работать корректно, но логи не будут записаны

Запуск приложения выполняется следующей командой:

```dotnet run --project "TaskManager"```

Если база данных на указанном сервере не существует, приложение создаст её при запуске. Также БД можно создать явно командой

```dotnet ef database update --project "TaskManager"```

Указанные команды необходимо выполнять из каталога решения. При выполнении из другого каталога нужно соответствующим образом изменить путь к проекту. При выполнении из каталога проекта параметр `--project` задавать не нужно

# Описание архитектуры
Основной интерфейс `ITaskRepository`: он позволяет просматривать список созданных задач, добавлять новые, удалять существующие, а также изменять статус выполнения указанной задачи

Подключение к хранилищу задач управляется и отображается с помощью классов `ConnectionView[Model]`, а отображение и работа со списком задач классами `TaskListView[Model]`

# Комментарии

### Интерфейс и требования ТЗ
По требованиям задачи должны быть представлены в виде таблицы, при этом добавление новой задачи должно быть организовано в виде добавления к этой таблице новой строки. 

Я считаю, что таблица как форма представления уместна, когда пользователю требуется воспринимать одно и то же свойство у нескольких сущностей, а при работе с задачами было бы достаточно простого списка.
Также добавление новой сущности через пустую строку в таблице может у некоторых пользователей создавать затруденения, ввиду неочевидности того, что очередная строка в таблице не отображает элемент, а позволяет добавлять новый. 

Я реализовал добавление задачи через отдельные элементы интерфейса

### Архитектура
Основная сущность - интерфейс `ITaskRepository`: он предоставлет доступ к существующим задачам; позволяет создавать новые, удалять существующие и изменять статус завершённости задачи

Подключение к базе данных обрабатывается и отображается `ConnectionView[Model]`, а работа со списком задач `TaskListView[Model]`

Об ошибках в приложении пользователь уведомляется с помощью уведомлений через интерфейс `IDesktoServides`
